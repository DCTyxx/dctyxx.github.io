理解socket

Socket 是应用层与TCP/IP的协议族通信的中间软件抽象层，他是一组接口。在设计模式中，Socket其实就是一个门面模式，把TCP/IP协议藏在Socket接口后面，医嘱简单的接口就是全部。

TCP协议与UDP协议

TCP：可靠的面向连接的协议、传输效率低全双工通信。
UDP：不可靠的、无连接的服务，传输效率高。

黏包现象

TCP粘包是指发送方发送的若干包数据到接收方接收时粘成一包，从接收缓冲区看，后一包数据的头紧接着前一包数据的尾。我们先来看一段代码。这是一段基于TCP的本地的远程操控代码，会出现黏包现象。UDP不会出现黏包现象但是会丢包，另一方面也反映了UDP的不稳定。
本质原因：首先只有在TCP协议中才会出现黏包现象，是因为TCP协议是面向流的协议，在发送的数据传输的过程中还有缓存机制来避免数据丢失，因此在连续发送小数据的时候以及接收大小不符的时候都容易出现黏包现象，黏包现象的本质：因为我们在接受数据的时候还不知道发送的数据长短。

解决黏包现象的方法

出现黏包现象的情况：

1、连续send两个小数据
2、两个recv，第一个recv特别小
本质：不知道要接受多大的数据。

解决方法：

首先，发送一下这个数据有多大，再按照数据的长度接收数据。

1、

优点：
1、确定到底接受了多大的数据；
2、要在文件中配置一个配置项：就是每一次recv的大小，buffer=4096;
3、当我们要发送大数据的时候，要明确告诉接受方要发送多大的数据，以便接收方能够准确的接收到所有的数据。
4、多用在文件传输的过程中：
大文件的传输，一定是按照字节读，每一次读固定的字节；
传输的过程中，一边读一边传，接收端，一边收一边写。

缺点：多了一次交互。
